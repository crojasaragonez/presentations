¿Que es?

  • Una arquitectura que buscan evitar tener servidores 24/7 encendidos.
  • Eliminan el tiempo oscioso de servidores.
  • Alternativas a patrones de diseño como working queue.
  • Busca reducir costos
  • El desarrollador solo tiene que pensar en el código y no en servidores ni infraestructura.

Conceptos Importantes

  • SaaS => Software as a service (Jira, Youtrack)
  • PaaS => Platform as a service (Heroku, Engine Yard, Azure)
  • IaaS => Infrastructure as a service (AWS, Azure)
  • BaaS => Backend as a service (Firebase, Parse(rip))
  • FaaS => Function as a service

FaaS

  • Nos permite tener una funcion "hosteada" en un proveedor tercero.
  • Se podría decir que es una mejora de RPC(Remote Procedure Call).
  • Corren en ambientes de alto desempeño.

Ventajas

  • Obtenemos muy buen rendimiento a menor costo.
  • Es tecnología verde porque ahorra horas de consumo de energía.

Desventajas

  • Dependencia de terceros.
  • Duplicación de código.
  • ¿Pruebas unitarias?.
