Información básica

¿Qué es?

Es un lenguaje de consultas para APIs web que se enfoca en la descripción y transmisión eficiente de datos atraves del protocolo http.

Historia

Las especificaciones de GraphQL fueron creadas en Facebook en el 2012 y puestas a prueba en las aplicaciones móviles. En el 2015 Facebook liberó las especificaciones al público y ahora está disponible en muchos lenguajes y frameworks de desarrollo web.

¿Porqué debería interesarme?

Graphql tiene ventajas en rendimiento y simplicidad comparado con otras alternativas clásicas como REST y SOAP. Además de ser usado por grandes compañías como Github, Twitter, Airbnb entre otros muchos más.

REST vs Graphql

Rest (CRUD de un vehículo.)

  • GET: /vehiculos[?color=rojo]
  • GET: /vehiculos/1
  • POST: /vehiculos
  • PUT | PATCH: /vehiculos/1
  • DELETE: /vehiculos/1

Rest (Detalles de un vehículo.)

  • GET: /vehiculos/1/propietario
  • GET: /vehiculos/1/mantenimientos
  • GET: /vehiculos/1/detalles

Graphql

type Query
type Mutation

Servidor

extend type Query {
    vehicles: [Vehicle!]!
    vehicle(id: ID!): Vehicle
}

type Vehicle {
    id: ID!
    plate_number: String!
    color: String!
    manufacturer: String!
    model: String!
    transmission: String!
    body_type: String!
    mileage: Float!
    year: String!
}

Cliente

query{
  vehicles{
    data{
      manufacturer
      mileage
    }
  }
}

Respuesta

{
  "data": {
    "vehicles": {
      "data": [
        {
          "manufacturer": "Nissan",
          "mileage": 868767
        },
        ...
      ]
    }
  }
}

Servidor

extend type Mutation {
    createVehicle(
        plate_number: String!
        color: String!
        manufacturer: String!
        model: String!
        transmission: String!
        mileage: Float!
        year: String!
    ): Vehicle! @create

    updateVehicle(
        plate_number: String!
        color: String!
        manufacturer: String!
        model: String!
        transmission: String!
        mileage: Float!
        year: String!
    ): Vehicle @update

    deleteVehicle(id: ID!): Vehicle @delete
}

Cliente

mutation{
  createVehicle(
    plate_number: "SDK-200"
    color: "Azul"
    manufacturer: "Chevrolet"
    model: "Aveo"
    transmission: "Manual"
    mileage: 60000
    year: "2014"
  ){
    id
  }
}

Respuesta

{
  "data": {
    "createVehicle": {
      "id": "101"
    }
  }
}

Ventajas

  • Menos código

    El código necesario para lograr operaciones CRUD es 0

  • Transmisión eficiente

    El cliente únicamente pide lo que nececita, nada mas, nada menos. Además de que puede combinar múltiples datos en una sola respuesta.

  • Documentación viva

    Los clientes pueden explorar por ellos mismos las capacidades del API y formar sus propias consultas de una manera muy flexible.
